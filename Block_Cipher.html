块加密

top

1.概述

所谓块加密就是每次加密一块明文，常见的加密算法有

  • IDEA 加密

  • DES 加密

  • AES 加密

块加盟是对称加密(加解密的密钥是一样的)

考虑到明文的长度的任意性，借助：

  • padding，即 padding 到指定分组长度

  • 分组加密模式，即明文分组加密的方式

2.基本策略

混淆

混淆，Confusion，将密文与密钥之间的统计关系变得尽可能复杂，使得攻击者即使获取了密文的一些统计特性，也无法推测密钥。一般使用复杂的非线性变换可以得到很好的混淆效果，常见的方法如下

  • S 盒

  • 乘法扩散

扩散，Diffusion，使得明文中的每一位影响密文中的许多位。常见的方法有

  • 线性变换

  • 置换

  • 移位，循环移位

3.基本结构

迭代结构

一般包括三个部分

  • 密钥置换

  • 轮加密函数

  • 轮解密函数

迭代加密的目的

  • 增加安全性：通过多个轮次的处理，使得密文与明文之间的关系更加复杂。每一轮都对数据进行进一步的“混淆”和“扩散”，使得攻击者难以通过简单的分析破解密钥或明文。

  • 对抗已知攻击：如差分攻击（Differential Cryptanalysis）和线性攻击（Linear Cryptanalysis）。迭代结构增加了破解的难度。

Feistel结构

加密

1.分割数据:

  • 对半分成（L，R）

2.迭代加密：

  • 轮函数（F）：将右半部分R作为输入，通过轮函数F与轮密钥进行加密，得到输出F(R, K)，其中K是该轮使用的轮密钥。

  • 异或操作（XOR）：将轮函数的输出与左半部分L进行异或操作，得到新的左半部分L'。

  • 交换：右半部分R保持不变，左半部分L'成为新的右半部分。然后，进入下一轮。

L' = R
R' = L XOR F(R, K)
Ln+1=RnRn+1=Ln⊕F(Rn,Kn)

 

3.迭代输出：

  • 得到m=(L,R)

解密

1.分割数据:

  • 对半分成（L，R）

2.迭代解密：

Rn=Ln+1Ln=Rn+1⊕F(Ln+1,Kn)

3.还原密文

 

Substitution-Permutation结构

加密

1.初始加密：

  • 将明文数据分为多个块，每个块的大小通常是固定的（例如，128位、256位等）。每个块通过代换和置换操作进行加密

2.每轮加密

每轮加密：

  • 代换操作（Substitution）：对数据块进行代换。通常使用S盒来对数据进行替换，使得每个输入值变为一个新的输出值。

  • 置换操作（Permutation）：对数据进行位的重新排列（置换）。这种操作增强了数据的扩散性，确保密文中的每个比特与明文中的多个比特相关。(或许用到P盒)

  • 轮密钥异或（XOR）：将当前的数据块与轮密钥进行异或操作，生成新的数据块。轮密钥通常是从原始密钥通过密钥扩展算法生成的。

3.最后一轮：

  • 最后一轮的处理与前几轮类似，但通常不包括置换操作（或者置换操作简化）。最后的输出就是加密后的密文。

密钥扩展

目前，密钥扩展的方法有很多，没有见到什么完美的密钥扩展方法，基本原则是使得密钥的每一个比特尽可能影响多轮的轮密钥。

ARX

top

1.概述

ARX 运算是如下 3 种基本运算的统称 - Add 有限域上的模加 - Rotate 循环移位 - Xor 异或

有许多常见的块加密算法在轮函数中只用到了这 3 种基本运算，典型例子如 Salsa20、Speck 等。另外 IDEA 也采用了类似的基本运算来构建加解密操作，不过以乘法代替了移位

2.优缺点

优点：运算简单，执行时间为常数，函数表达能力强

缺点：Rotate、Xor 对于单个 bit 来说均是完全线性的运算，可能会带来一定的脆弱性

DES

1.基本介绍

Data Encryption Standard(DES)，数据加密标准，是典型的块加密，其基本信息如下

  • 输入 64 位。

  • 输出 64 位。

  • 密钥 64 位，使用 64 位密钥中的 56 位，剩余的 8 位要么丢弃，要么作为奇偶校验位。

  • Feistel 迭代结构

    • 明文经过 16 轮迭代得到密文。

    • 密文经过类似的 16 轮迭代得到明文

4.f函数介绍

在此之前，引用一句话“更深入地理解Feistel密码的工作原理，下面这个解释非常有用：将f函数看作是有两个输入参数Ri和ki的伪随机数产生器。该伪随机数产生器的输出将使用XOR操作来加密左半部分Li。从第2章可知，如果攻击者不能预测出f函数的输出，则说明此加密方法是非常强壮的。”

在第i轮中，f的输入为Ri,32位和ki，56位，把R_i分成8组4位的分组，利用E-盒进行扩展到48位，即4位=>6位，然后和eki异或，所得的结果放入8个不同的S-盒映射到8组4位的32位数

5.密钥编排

密钥编排从原始的56位密钥中得到16个轮密钥ki，其中每个轮密钥ki都是48位。轮密钥的另一个术语叫子密钥。首先请注意，DES的输入密钥通常是64位，其中每第8个位都作为前面7位的一个奇校验位。没有人清楚以这种方式规范DES的原因。任何情况下，这8个奇校验位都不是真的密钥位，也没有增加密码的安全性。所以可以说DES是一个56位的密码，而不是64位的。

首先，置换选择1(PC-1),将64位=>56位，去除每第八位(奇偶校验位)

得到的56位密钥将分为C0和D0两部分，而图3-12显示了实际密钥编排开始时的操作。长度均为28位的左右两部分将周期性地向左移动一位或两位(即循环移位)，而移动的具体位数则取决于轮数i，其规则如下：

在第i=1, 2, 9, 16轮中，左右两部分向左移动一位。 在i≠1, 2, 9, 16的其他轮中，左右两部分向左移动两位。 注意，循环移位要么发生在左半部分，要么发生在右半部分。循环移动位置的总数为4∗1+12∗2=28。这将带来一个有趣的属性，即C0=C16和D0=D16。这对解密密钥编排(其中子密钥都是逆序生成的)非常有用。

from Crypto.Cipher import DES
from Crypto.Util.Padding import pad, unpad
from Crypto.Random import get_random_bytes


# 生成一个随机的8字节(64位)密钥
key = get_random_bytes(8)


# 创建一个DES cipher对象
cipher = DES.new(key, DES.MODE_ECB)  # 使用ECB模式，实际应用中建议使用更安全的模式如CBC


# 要加密的数据
data = b'Hello, DES!'


# 加密数据
padded_data = pad(data, DES.block_size)  # 填充数据以匹配块大小
ciphertext = cipher.encrypt(padded_data)
print("Ciphertext:", ciphertext.hex())


# 解密数据
decipher = DES.new(key, DES.MODE_ECB)
plaintext = unpad(decipher.decrypt(ciphertext), DES.block_size)
print("Plaintext:", plaintext.decode())
